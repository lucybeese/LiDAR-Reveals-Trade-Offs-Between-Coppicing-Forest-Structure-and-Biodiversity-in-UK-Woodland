# LiDAR Reveals Complex Trade-Offs Between Coppicing, Forest Structure, and Biodiversity in UK Woodland Restoration

**NOTE:** The associated PhD thesis chapter, manuscript and supp info will be uploaded following submission of the thesis in **April 2026**.

This repository contains the workflows for **data processing, analysis, and figure generation** supporting the study:

**Beese, L. V. J. & Jucker, T. (2026).**  
*LiDAR Reveals Complex Trade-Offs Between Coppicing, Forest Structure, and Biodiversity in UK Woodland Restoration.*

This study integrates **airborne LiDAR data, citizen science biodiversity surveys (EarthTrack), and long-term dormouse monitoring records** to quantify how **coppicing-with-standards influences forest structure, plant biodiversity, and species-specific habitat use**.

---

# Study overview

Coppicing is widely implemented in UK woodland management to enhance biodiversity and support species of conservation concern, particularly the hazel dormouse (*Muscardinus avellanarius*). However, the ecological consequences of coppicing are complex, with potential trade-offs between **forest structure, plant biodiversity, and species-specific habitat suitability**.

This study evaluates these interactions within a coppiced woodland in North Wales by combining **LiDAR-derived structural metrics** with **ground-based biodiversity observations** and **dormouse abundance records**.

Specifically, the study examines:

- **How coppicing influences forest canopy structure over time**
- **Whether plant biodiversity varies predictably with time since coppicing**
- **How dormouse abundance responds to vegetation structure and composition**
- **Whether LiDAR-derived structural and topographic metrics can act as proxies for biodiversity and species abundance**

By integrating remote sensing and ecological field data, this work provides a framework for **scaling biodiversity monitoring and evaluating restoration effectiveness in managed woodlands**.

# Code overview

The `code/` directory contains scripts used to:

- Process and integrate:
  - **LiDAR-derived canopy and terrain metrics**
  - **EarthTrack biodiversity data**
  - **Dormouse monitoring records (NDMP)**

- Derive structural variables:
  - Canopy height, canopy volume  
  - Gap proportion  
  - Leaf Area Index (LAI)  
  - Vertical Complexity Index (VCI)  
  - LiDAR intensity  

- Derive topographic variables:
  - Elevation / HALP  
  - Slope, aspect (cosine transformed)  
  - Curvature metrics and flow direction  

- Fit statistical models:
  - **GAMs** for non-linear structural responses  
  - **Gamma GLMs** for biodiversity  
  - **Negative binomial GLMs** for dormouse abundance  
  - **Linear models** for LiDAR–biodiversity relationships  

- Evaluate:
  - Multicollinearity (VIF-based selection)  
  - Model fit (AIC, deviance explained)  
  - Residual diagnostics (e.g. DHARMa)  

- Generate:
  - Figures for main results  
  - Supplementary tables and diagnostics  

Scripts are organised sequentially to reflect the analytical workflow.

# Data dictionary (key variables)

| Variable | Description |
|----------|-------------|
| **LAI** | Leaf Area Index derived from LiDAR |
| **VCI** | Vertical Complexity Index |
| **Gap proportion** | Proportion of canopy gaps (threshold-based) |
| **Intensity** | LiDAR return intensity |
| **HALP** | Height Above the Lowest Point (topographic proxy) |
| **Mean Shannon Index** | Plot-level plant diversity metric |
| **Mean Dormice Abundance** | Mean dormouse counts per plot |
| **Years since coppicing** | Time since last management intervention |
| **No. of ET points** | Number of EarthTrack samples (sampling effort) |
| **No. of DM records** | Dormouse survey effort |

# Reproducibility

All analyses were conducted in **R**.

Key packages include:

- `mgcv` (GAMs)  
- `MASS` (negative binomial GLMs)  
- `lme4`, `lmerTest` (mixed models where applicable)  
- `DHARMa` (residual diagnostics)  
- `terra`, `lidR`, `leafR` (LiDAR processing)  
- `tidyverse`, `data.table` (data handling)  

Users should ensure package versions are compatible with those used during analysis.
