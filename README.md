# LiDAR Reveals Complex Trade-Offs Between Coppicing, Forest Structure, and Biodiversity in UK Woodland Restoration

This repository contains the data processing, analysis, and figure-generation code supporting the study:

**Beese, L. V. J. & Jucker, T. (2026).**  
*LiDAR Reveals Complex Trade-Offs Between Coppicing, Forest Structure, and Biodiversity in UK Woodland Restoration.*

The study combines airborne LiDAR–derived structural metrics with ecological survey data to examine how coppicing management influences forest structure and biodiversity responses in UK woodlands.

---

## Repository structure

    ├── code/               # Analysis and modelling scripts
    ├── data/               # Raw and processed data (see Data availability)
    ├── figures/            # Figures generated by the analysis
    ├── manuscript/         # Manuscript files (drafts / submitted versions)
    ├── supp_info/          # Supplementary information
    ├── README.md
    └── .gitignore

---

## Code overview

The `code/` directory contains scripts used to:

- Process airborne LiDAR data and derive forest structural metrics  
- Integrate LiDAR outputs with biodiversity and management data  
- Fit statistical models linking coppicing, structure, and biodiversity responses  
- Generate figures and summary tables used in the manuscript  

Scripts are intended to be run sequentially. Where applicable, filenames indicate execution order.

---

## Data availability

Some raw LiDAR point clouds and ecological datasets are **not included** in this repository due to data size, licensing, and access restrictions.

The `data/` directory contains:
- Placeholder files and README notes describing data sources and preprocessing steps  
- Derived metrics and cleaned datasets sufficient to reproduce analyses and figures, where redistribution is permitted  

Please contact the author for guidance on accessing restricted datasets.

---

## Reproducibility

Analyses were conducted primarily in **R**.

Users should ensure that package versions are compatible with those current at the time of analysis. An `renv` environment may be added in future updates to improve reproducibility.

---

## How to run the analysis

1. Clone the repository  
2. Install required R packages  
3. Populate the `data/` directory according to the notes provided  
4. Run scripts in the `code/` directory  
5. Outputs (figures and tables) will be written to `figures/` and relevant output folders
